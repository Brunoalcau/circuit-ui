# Message

Messages are used to serve our merchants with informative snippets or feedback
on a given action. All banners should be dismissible with an "x", unless they
require an essential action to be taken.

<Playground>
  <Message>
    <MessageIcon type={MessageIcon.SUCCESS}/>
    <Heading size={Heading.KILO} element="h4" margin={false}>
      Transaction successfully refunded
    </Heading>
  </Message>
</Playground>

## Best practices

Messages can be categories into three groups:

1. **Feedback Messages:** Used to provide feedback to a merchant after an action
   has been taken. These notifications disappear automatically.
2. **Informative Messages:** Can contain information which isn't essential but may
   be useful to the merchant. (e.g. New feature announcement) It is recommended
   to use a call to action.
3. **Attention Messages:** When action is required by the merchant, attention
   messages should be used and should clearly highlight which action is
   required. **A call to action must be displayed.**

## Usage guidelines

- **Do** allow merchants to dismiss all messages unless immediate action is required.
- **Do** direct merchants to the relevant place with a CTA if action is required.
- **Do** use feedback messages to validate an action has been completed.
- **Do** use the correct badge colour to signify sentiment of a message.
- **Do not** use attention messages unless a vital action is required.

## Notification messages

Messages are generally displayed as notifications to merchants.

### Notification List

You can display a list of messages with `NotificationList`.

<Playground>
  <NotificationList>
    <Message>
      <MessageIcon type={MessageIcon.SUCCESS}/>
      <Heading size={Heading.KILO} element="h4" noMargin>
        Account created
      </Heading>
    </Message>
    <Message>
      <MessageIcon type={MessageIcon.SUCCESS}/>
      <Heading size={Heading.KILO} element="h4" noMargin>
        Transaction successfully refunded
      </Heading>
    </Message>
  </NotificationList>
</Playground>
